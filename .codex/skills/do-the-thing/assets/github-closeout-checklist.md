# GitHub Closeout Checklist

When the selected issue is truly complete:

1. Post a substantive completion note on the child issue.
2. Close the child issue.
3. Update the EPIC so the child issue is checked off.
4. Add a brief EPIC progress note if useful.

## Child issue note must include
- summary of what was implemented
- tests added or updated
- validation commands run
- adversarial review findings
- corrections made after review
- any intentionally deferred items that are outside issue scope

## EPIC update should include
- the completed child issue marked done
- current progress clarity if the EPIC body is acting as the main tracker

## Completion standard
Do not close the child issue until:
- tests pass
- acceptance criteria pass
- adversarial review has been performed
- discovered defects have been corrected
