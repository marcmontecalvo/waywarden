# Required Output Format

Return exactly these sections, in this order.

## SOURCE MATERIAL
- files/docs used
- short summary of what they describe

## EPIC DRAFT
- title
- goal
- ordered checklist
- global constraints
- definition of done

## CHILD ISSUES
For each child issue, provide:
- issue id/title
- goal
- scope
- non-goals
- acceptance criteria
- required tests
- blocking dependencies

## AMBIGUITIES OR MISSING DECISIONS
- bullet list
- write `none` if none

## RECOMMENDED ORDER
- ordered list of issue ids with one-line rationale each

## RESULT
- `PASS` if the issue set is specific enough for reliable implementation
- `FAIL` if major ambiguity remains
