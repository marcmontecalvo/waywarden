# Notes on sourcing

These research notes are intentionally **pattern-oriented**.

For proprietary products such as Wispr Flow, Manus, and Claude-related experiences, the purpose of these docs is to capture:
- the product qualities worth emulating
- the architectural implications for WayWarden
- the likely OSS implementation direction

They are **not** intended to be exhaustive or treated as vendor-authored specs.

When a research reference graduates into a real roadmap item, create a follow-up doc under architecture or issues that:
- narrows the scope
- identifies candidate OSS dependencies
- defines safety and approval boundaries
- specifies what will actually be built in-repo
